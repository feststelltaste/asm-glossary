# Domain-Driven Refactoring

> Restructuring code so that its shape reflects the domain it models, grouping related concepts, naming things after the business language, and separating core domain logic from infrastructure concerns. For agents, a domain-aligned codebase is dramatically easier to navigate: names carry meaning, boundaries are visible, and retrieval (via RAG or code knowledge graphs) returns more relevant chunks. Domain-driven refactoring is therefore both a goal of modernization and a prerequisite for effective agentic work on the codebase.
