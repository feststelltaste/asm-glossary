# Inkrementelle Verbesserung

> Die Praxis, Legacy-Systeme durch viele kleine, rücknehmbare Änderungen zu modernisieren, statt durch eine große Komplettumschreibung. Jeder Schritt hinterlässt das System in einem lauffähigen Zustand und lässt sich unabhängig verifizieren. Inkrementelle Verbesserung passt natürlich zu Agenten: Kürzere Schritte bedeuten kleineren Kontext, kleinere Diffs, schnelleres Feedback durch Tests und einfachere menschliche Prüfung. Sie begrenzt außerdem den Schadensradius, falls der Agent einen Fehler macht, und hält so die menschliche Aufsicht handhabbar.
