# Spec-Driven Development

> Eine Entwicklungsmethodik, bei der detaillierte, formale Spezifikationen als Single Source of Truth dienen und Code ein daraus abgeleitetes, generiertes Artefakt ist. Statt Code direkt zu schreiben, verfassen Ingenieure Spezifikationen, die Verhalten, Constraints und Grenzfälle beschreiben. KI-Agenten generieren, validieren und pflegen dann die Implementierung gegen diese Spezifikationen. Die Spezifikation ersetzt den Code als primäres Engineering-Artefakt und macht die Absicht explizit und nachvollziehbar, selbst wenn der resultierende Code nie von einem Menschen gelesen wird.
