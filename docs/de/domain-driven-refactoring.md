# Domain-Driven Refactoring

> Die Codebasis so umstrukturieren, dass ihre Form die modellierte Domäne widerspiegelt: Zusammengehörige Konzepte werden gruppiert, Dinge nach der Fachsprache benannt, und Kernlogik wird von Infrastrukturbelangen getrennt. Für Agenten ist eine domänenorientierte Codebasis deutlich leichter zu navigieren: Namen tragen Bedeutung, Grenzen sind sichtbar, und Retrieval (über RAG oder Code Knowledge Graphs) liefert relevantere Ergebnisse. Domain-Driven Refactoring ist damit zugleich Ziel der Modernisierung und Voraussetzung für wirksame agentische Arbeit an der Codebasis.
