# Feature Toggle

> Ein Mechanismus, der es ermöglicht, neue oder modernisierte Funktionalität auszuliefern, ohne sie sofort für Nutzer sichtbar zu machen. Indem Änderungen hinter einem Toggle gekapselt werden, können Teams Code kontinuierlich mergen und deployen, während die Aktivierung unabhängig vom Deployment gesteuert wird. Das entkoppelt Release von Deployment und reduziert das Risiko großflächiger Rollouts. Bei agentengestützter Modernisierung erlauben Feature Toggles dem Agenten, Funktionalität Schritt für Schritt zu migrieren, während das alte Verhalten als sofortiger Fallback verfügbar bleibt.
