# Halluzination

> Wenn ein Agent plausibel aussehenden, aber falschen Code generiert, z.B. nicht existierende APIs aufruft, Geschäftslogik falsch interpretiert oder Abhängigkeiten erfindet. Das Hauptrisiko bei agentengestützter Modernisierung, weil halluzinierter Code kompilieren und oberflächlich funktionieren kann. Test Harnesses und Code Knowledge Graphs sind die wichtigsten Gegenmaßnahmen.
