# Slicing

> Zerlegung eines Systems entlang seiner fachlichen Grenzen in dünne, vertikale Scheiben, die jeweils einen eigenständigen Bereich fachlicher Funktionalität abbilden. Jede Scheibe isoliert einen zusammenhängenden Teil der Domäne, der unabhängig modernisiert, getestet und ausgeliefert werden kann. Feature Toggles können das Rollout einzelner Scheiben absichern, und Workload Design bestimmt, ob eine Scheibe von einem Agenten, einem Menschen oder einer Kombination bearbeitet wird. Slicing ist besonders wirksam für agentengestützte Arbeit, weil ein gut abgegrenzter Schnitt in ein einzelnes Kontextfenster passt und in einer Iteration des agentischen Zyklus abgeschlossen werden kann.
