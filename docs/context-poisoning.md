# Context Poisoning

> The introduction of incorrect, outdated, or misleading information into an agent's working context, whether from stale documentation, deprecated APIs, or legacy code patterns. A poisoned context biases the agent toward reproducing existing flaws instead of correcting them. Unlike hallucination, where the agent invents facts, context poisoning causes the agent to faithfully follow bad input. Codebase conditioning and explicit architectural examples in the prompt are key countermeasures.
