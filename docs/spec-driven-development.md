# Spec-Driven Development

> A development methodology where detailed, formal specifications serve as the source of truth, and code is a generated artifact derived from them. Instead of writing code directly, engineers write specifications that describe behavior, constraints, and edge cases. AI agents then generate, validate, and maintain the implementation against those specs. The specification replaces code as the primary engineering artifact, making intent explicit and traceable even when the resulting code is never read by a human.
