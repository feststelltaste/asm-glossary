# Horseshoe Model

> A modernization approach that works in three phases: extracting an abstract representation from legacy code, transforming that representation to meet new requirements, and generating a fresh implementation from the result. The horseshoe model avoids direct code-to-code migration by operating at a higher level of abstraction, where structural and design decisions are easier to reason about. Related approaches include clean room rewrite (which uses a specification as the intermediate representation) and idiomatic transpilation (which uses an AST).
