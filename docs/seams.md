# Seams

![Seams](assets/images/en/seams.png)

> A legacy-code concept describing a place in the code where behavior can be changed without editing the code at that location, typically by substituting a collaborator, object, or function at a boundary. Seams make legacy code testable and modifiable in small, local steps. For agents, seams are essential: they turn hard-to-reason-about code into units that can be isolated, exercised, and changed one at a time, shrinking the scope of what an agent needs to understand before making a change.
