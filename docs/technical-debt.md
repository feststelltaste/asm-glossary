# Technical Debt

> The long-term costs that arise when teams take shortcuts in implementation, consciously or not: messy code, missing tests, architectural compromises. Technical debt lives in the code and makes systems harder to change. It is the most visible and best understood form of debt, with established practices like refactoring, code review, and TDD. In the context of agent-assisted modernization, AI can actively reduce technical debt while simultaneously accelerating cognitive and intent debt.
