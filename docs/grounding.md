# Grounding

> The practice of anchoring an agent's output in verified, authoritative sources such as existing code, test results, and documentation rather than allowing it to rely on pattern completion alone. Grounding reduces hallucinations and context poisoning by ensuring the agent references actual project artifacts when generating or transforming code.
